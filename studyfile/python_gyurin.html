1. 주제 선정 이유¶

한국은 배달 문화가 매우 잘 되어 있고, 스마트폰의 보급율이 매우 높은 나라로서 음식 배달 어플이 굉장이 잘 활성화되어 있습니다.

음악 어플에서는 사용자의 음악 취향에 맞게 음악을 추천해주고, '유튜브'와 같은 영상 어플이나 '넷플릭스' 등의 영화 어플에서는 사용자의 취향에 맞게 동영상이나 영화를 추천해줍니다.

하지만 '배달의 민족' 혹은 '요기요'와 같은 음식 배달 어플에서는 사용자의 취향에 맞게 음식점을 추천해주는 기능이 없습니다.

음악과 영화가 사람에 따라 호불호가 갈리는 것 못지 않게 입맛도 사람마다 천차만별이기에 저는 배달 어플 사용자의 주문 내역이나 검색 이력을 바탕으로 사용자가 좋아할만한 음식점을 추천해주는 프로그램을 만들어보기로 하였습니다.

2. 가설 정의¶

입력 값은

  1. 사용자가 배달음식을 받고 싶어하는 곳의 주소 이름
  2. 사용자가 주문하려는 시간
  3. 사용자의 취향을 알 수 있는 요소들 (검색내역과 주문내역의 파일)

결과 값은

  1. 사용자가 좋아할만한 음식점 : 사용자의 취향에 맞는 음식점을 추천하기 위해 음식점을 다양한 방법으로 세분화해야 하는데, 우선 음식의 종류(한식, 분식, 중식, 치킨 등)로 구분할 수도 있고 프랜차이즈의 여부로도 구분할 수 있습니다. 사용자의 주문 내역과 검색 내역과 연관된 음식점들을 종합해 분류하고, 입력 받은 주소와 함께 고려해 가장 높은 비율을 차지하는 음식점 분야에 있는 매장들을 추천합니다.
  2. 갈 수 있는 음식점 : 사용자가 좋아할만한 음식 분야와 지역에 대한 정보를 바탕으로 음식점을 찾은 후, 인터넷 사이트에 있는 각 매장 정보에 기재되어 있는 운영시간에 대한 정보를 바탕으로 갈 수 있는 음식점이 가려집니다.

최종적으로 도출할 내용은 사용자가 원하는 시간(현재 시간 or 새로 입력한 시간)에 갈 수 있는 사용자 취향의 음식점들입니다.

#7. function_change_time_to_number(giventime):¶

(1) 목적:

이는 '오전(or 오후) ?시 ?분'의 str type을 입력 parameter로 받고 이를 천의 자리와 백의 자리 숫자는 '시', 십의 자리와 일의 자리 숫자는 '분'을 의미하는 정수 값으로 바꾸고 그 값을 반환하는 함수입니다.

(2) 방법:

  1. 오전/ 오후에 대해서는 오후라면 12시간을 더한 셈 쳐서, 즉 천의 자리 숫자에 1, 백의 자리 숫자에 2를 더해준 셈 쳐서 result에 1200을 더해줍니다.
  1. 입력 parameter인 giventime을 쪼개준 A라는 list가 길이가 3이라면, 이는 시 값과 분 값을 모두 가졌음을 의미합니다. '10시','30분' 과 같은 str type을 숫자 (int)로 바꿔줄려고 합니다. 일단 어짜피 어쨋던간에 '오후'나 '오전'은 '시'나 '분'이라는 str을 포함하지 않습니다. A라는 list의 두번째 값인 시 값에서 '시'를 제거해줍니다. 그리고 A라는 list의 세번째 값인 분 값에서 '분'이라는 글자, 즉 str을 제거해줍니다. result는 이제 쉽게 구할 수 있습니다.
  1. A의 길이가 2라면, 이는 시 값만 가졌음을 의미하므로 A라는 list의 두번째 값인 시 값에서 '시'를 제거해주고 result를 바로 구해줍니다.

    제가 모든 음식점 사이트들의 경우에 대해 조사하는 것은 무리가 있을 것 같으니, 이건 애초에 사이트 측에서 근본적으로 음식점 장사 시간 정보에 대해 체계적으로 구현되어야 할 것 같습니다.

  • 문제점 4 :

    사이트에 정보가 기재되어 있지 않을 때, 즉 findAll로 정보를 찾을 수 없을 때 error가 나지 않고 그냥 무시하고 넘어가게 하고 싶었습니다. 그래서 정보가 있는지 없는지 boolean으로 반환하는 함수가 있다면 그걸 쓰려 했는데, 제가 찾지 못한건지, 그런 함수는 없는건지 그렇게 할 수 없었습니다. 이는 마찬가지로 웹프로그래밍에 대해 더 배워야 해결할 수 있을 것 같습니다.

  • 문제점 5 :

    문제점 3과 마찬가지로 사이트 측에서 근본적으로 음식점 장사 시간 정보에 대해 체계적으로 구현되어야 할 것 같습니다.

NAME
    urllib.request - An extensible library for opening URLs using a variety of protocols

DESCRIPTION
    The simplest way to use this module is to call the urlopen function,
    which accepts a string containing a URL or a Request object (described
    below).  It opens the URL and returns the results as file-like
    object; the returned object has some extra methods described below.
    
    The OpenerDirector manages a collection of Handler objects that do
    all the actual work.  Each Handler implements a particular protocol or
    option.  The OpenerDirector is a composite object that invokes the
    Handlers needed to open the requested URL.  For example, the
    HTTPHandler performs HTTP GET and POST requests and deals with
    non-error returns.  The HTTPRedirectHandler automatically deals with
    HTTP 301, 302, 303 and 307 redirect errors, and the HTTPDigestAuthHandler
    deals with digest authentication.
    
    urlopen(url, data=None) -- Basic usage is the same as original
    urllib.  pass the url and optionally data to post to an HTTP URL, and
    get a file-like object back.  One difference is that you can also pass
    a Request instance instead of URL.  Raises a URLError (subclass of
    OSError); for HTTP errors, raises an HTTPError, which can also be
    treated as a valid response.
    
    build_opener -- Function that creates a new OpenerDirector instance.
    Will install the default handlers.  Accepts one or more Handlers as
    arguments, either instances or Handler classes that it will
    instantiate.  If one of the argument is a subclass of the default
    handler, the argument will be installed instead of the default.
    
    install_opener -- Installs a new opener as the default opener.
    
    objects of interest:
    
    OpenerDirector -- Sets up the User Agent as the Python-urllib client and manages
    the Handler classes, while dealing with requests and responses.
    
    Request -- An object that encapsulates the state of a request.  The
    state can be as simple as the URL.  It can also include extra HTTP
    headers, e.g. a User-Agent.
    
    BaseHandler --

    import urllib.request
    
    # set up authentication info
    authinfo = urllib.request.HTTPBasicAuthHandler()

        Open the URL url, which can be either a string or a Request object.
        
        *data* must be an object specifying additional data to be sent to
        the server, or None if no such data is needed.  See Request for
        details.
        
        urllib.request module uses HTTP/1.1 and includes a "Connection:close"
        header in its HTTP requests.
        
        The optional *timeout* parameter specifies a timeout in seconds for
        blocking operations like the connection attempt (if not specified, the
        global default timeout setting will be used). This only works for HTTP,
        HTTPS and FTP connections.
        
        If *context* is specified, it must be a ssl.SSLContext instance describing
        the various SSL options. See HTTPSConnection for more details.
        
        The optional *cafile* and *capath* parameters specify a set of trusted CA
        certificates for HTTPS requests. cafile should point to a single file
        containing a bundle of CA certificates, whereas capath should point to a
        directory of hashed certificate files. More information can be found in
        ssl.SSLContext.load_verify_locations().
        
        The *cadefault* parameter is ignored.
        
        This function always returns an object which can work as a context
        manager and has methods such as
        
        * geturl() - return the URL of the resource retrieved, commonly used to
          determine if a redirect was followed
        
        * info() - return the meta-information of the page, such as headers, in the
          form of an email.message_from_string() instance (see Quick Reference to
          HTTP Headers)
        
        * getcode() - return the HTTP status code of the response.  Raises URLError
          on errors.
        
        For HTTP and HTTPS URLs, this function returns a http.client.HTTPResponse
        object slightly modified. In addition to the three new methods above, the
        msg attribute contains the same information as the reason attribute ---
        the reason phrase returned by the server --- instead of the response
        headers as it is specified in the documentation for HTTPResponse.
        
        For FTP, file, and data URLs and requests explicitly handled by legacy
        URLopener and FancyURLopener classes, this function returns a
        urllib.response.addinfourl object.

        the default installed global OpenerDirector uses UnknownHandler to ensure
        this never happens).

        environment variable like http_proxy is set), ProxyHandler is default
        installed and makes sure the requests are handled through the proxy.

        Retrieve a URL into a temporary location on disk.

        the temporary file location. The reporthook argument should be
        a callable that accepts a block number, a read size, and the

        the following reserved characters.
